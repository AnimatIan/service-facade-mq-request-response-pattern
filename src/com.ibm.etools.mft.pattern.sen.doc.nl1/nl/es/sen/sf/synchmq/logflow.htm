Subflujo Log para el patrón Fachada de servicio a WebSphere MQ: solicitud-respuesta

El registro de mensajes en una base de datos no se realiza en los flujos de mensajes que manejan transacciones empresariales. Por el contrario, los mensajes persistentes se escriben en una cola de WebSphere MQ, que se procesa por separado. Escribir mensajes persistentes en una cola de WebSphere MQ aísla cualquier efecto en el rendimiento de las actualizaciones de base de datos, con lo que se permite el control de dónde y cuándo se procesan las colas.

El patrón supone que se añaden datos de registro a la cabecera MQRFH2 si hay una. Si no existe ninguna cabecera MQRFH2, se crea una. El cuerpo del mensaje se registra cuando llega al flujo de mensajes. Se han añadido los siguientes datos de registro:

  • Indicación horaria del registro
  • Nombre del intermediario
  • Nombre del flujo de mensajes
  • Estado (0 éxito)
  • Modalidad - Solicitud / Respuesta
