Tareas que deben completarse antes de aplicar el patrón Fachada de servicio a WebSphere MQ: solicitud-respuesta

Antes de aplicar el patrón para producir una instancia de patrón, debe determinar cuáles de las tareas siguientes son necesarias. Si es preciso, realice las tareas siguientes:

  • Identifique que este patrón cumple el requisito empresarial y puede ser soportado por las aplicaciones solicitante y proveedor.
  • Defina o identifique la interfaz del proveedor y la interfaz de servicio asociada.
  • Asigne un nombre a la instancia de patrón. Identifique el fin de la aplicación de patrón para que pueda decidir cuál es el nombre adecuado.
  • Seleccione colas y gestores de colas de WebSphere MQ. Las colas y los gestores de colas existentes pueden utilizarse, por ejemplo, para acceder a la aplicación de proveedor y debe elegir sus nombres.
  • Identifique cualquier cola y gestor de colas adicionales que este patrón necesite, y decida qué nombres desea utilizar. Tenga en cuenta las políticas, estándares o convenios de denominación que deben aplicarse.
  • Decida si necesita un registro. Si el registro es necesario, debe decidir a qué gestor de colas y cola se enviarán los mensajes de registro. Puede enviar mensajes de registro a una cola de registro general que utilicen varios flujos de mensajes, o puede utilizar una cola específicamente para la instancia de patrón.
  • Decida si necesita mensajes de error.
  • Decida si necesita validación.
  • Cree WSDL desplegable en un proyecto de conjunto de mensajes.
