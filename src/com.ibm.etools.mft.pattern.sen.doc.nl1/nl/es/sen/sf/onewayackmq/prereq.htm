Tareas que deben completarse antes de aplicar el patrón Fachada de servicio a WebSphere MQ: unidireccional con acuse de recibo

Antes de aplicar el patrón para producir una instancia de patrón, debe determinar cuáles de las tareas siguientes son necesarias. Si es preciso, realice las tareas siguientes:

  • Compruebe que este patrón cumple los requisitos empresariales y puede ser soportado por las aplicaciones solicitante y proveedor.
  • Seleccione un nombre exclusivo para la instancia de patrón. Puede utilizar el patrón varias veces y crear varias instancias distintas, que pueden distinguirse por sus nombres. Identifique el fin de la instancia de patrón para que pueda decidir un nombre adecuado o significativo.
  • Defina o identifique la interfaz del proveedor y la interfaz de servicio asociada. Cree WSDL desplegable en un proyecto de conjunto de mensajes.
  • Identificar las colas y los gestores de colas. Las colas y los gestores de colas existentes pueden utilizarse, por ejemplo, para acceder a la aplicación de proveedor y es preciso elegir sus nombres. Identifique cualquier cola y gestor de colas adicionales que este patrón necesite, y decida qué nombres desea utilizar. Tenga en cuenta las políticas, estándares o convenios de denominación que deben aplicarse.
  • Decida si necesita un registro de mensajes de solicitud. Si el registro es necesario, debe decidir a qué gestor de colas y cola se enviarán los mensajes de registro. Puede enviar mensajes de registro a una cola de registro general que utilicen varios flujos de mensajes, o puede utilizar una cola específicamente para la instancia de patrón.
  • Decida si desea producir mensajes de error para la instancia de patrón. En este patrón, los mensajes de errores sirven para notificar errores en el proceso, no para conservar los datos. Puede enviar mensajes de error a una cola de errores generales que utilizan varios flujos de mensajes, o puede utilizar una cola específicamente para la instancia de patrón. Si elige enviar mensajes de error en la instancia de patrón, puede activar o desactivar esta función configurando el archivo archivador de intermediario (BAR) antes del despliegue.
