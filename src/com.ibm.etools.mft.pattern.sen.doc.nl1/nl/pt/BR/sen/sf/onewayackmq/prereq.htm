Tarefas a serem concluídas antes de aplicar o padrão de Service Facade para WebSphere MQ: unidirecional com reconhecimento

Antes de aplicar o padrão para produzir uma instância padrão, é necessário determinar quais das tarefas a seguir são necessárias. Se necessário, conclua as tarefas a seguir:

  • Verifique se este padrão atende seus requisitos de negócios e se podem ser suportados pelos aplicativos do solicitante e do provedor.
  • Selecione um nome exclusivo para a instância padrão. É possível usar o padrão várias vezes e criar múltiplas instâncias diferentes, que podem ser distinguidas por seus nomes. Identifique o propósito da instância padrão para que você possa decidir sobre um nome apropriado ou significativo.
  • Defina ou identifique a interface do provedor e a interface de serviço associada. Crie o WSDL implementável em um projeto do conjunto de mensagens.
  • Identifique filas e gerenciadores de filas. Os gerenciadores de filas e filas existentes podem ser usados, por exemplo, para acessar o aplicativo do provedor e seus nomes devem ser decididos. Identifique quaisquer gerenciadores de filas e filas adicionais que sejam necessários para este padrão e decida sobre seus nomes. Leve em conta quaisquer políticas, normas ou convenções de nomenclatura que devem ser aplicadas.
  • Decida se você requer a criação de log de mensagens de pedido. Se a criação de log for necessária, é preciso decidir para qual gerenciador de filas e fila as mensagens de criação de log são enviadas. É possível enviar mensagens de criação de log para uma fila de log geral que é usada por múltiplos fluxos de mensagens ou você pode usar uma fila especificamente para a instância padrão.
  • Decida se você deseja produzir mensagens de erro para a instância padrão. Neste padrão, as mensagens de erro são para notificação de erros sobre processamento, não para preservação de dados. É possível enviar mensagens de erro para uma fila de erros geral que é usada por múltiplos fluxos de mensagens ou você pode usar uma fila especificamente para a instância padrão. Se escolher enviar mensagens de erro na instância padrão, é possível ativar esta função ou desativá-la configurando o arquivo broker archive (BAR) antes da implementação.
