Padrão do Service Facade para WebSphere MQ: unidirecional com reconhecimento

Use o padrão de Service Facade para WebSphere MQ: unidirecional com reconhecimento para apresentar uma interface de serviço da Web aos clientes e preencher as solicitações de serviço usando um aplicativo ativado para WebSphere MQ.

Use este padrão para ligar os protocolos HTTP assíncronos e protocolos do sistema de mensagens confiáveis para manipular atualizações com uma garantia de que os pedidos serão salvos para processamento.

Este padrão fornece loose coupling entre solicitantes e provedores na sincronização, nos protocolos e no transporte. Ele é apropriado para as interfaces de serviço em sistemas existentes.

Solução

A solução é implementar um fluxo de mensagens que fornece um ponto de entrada de serviço. Quando uma solicitação de serviço da Web é recebida via HTTP, os dados são colocados como uma mensagem persistente em uma fila para o aplicativo do provedor. Quando a mensagem é colocada, um reconhecimento é retornado para confirmar que a solicitação foi aceita. Se um cliente receber um reconhecimento, a mensagem foi gravada na fila. Se uma falha for recebida, nenhuma mensagem foi enviada, entretanto, se a falha for uma expiração, o caso é indeterminado, mas aplicativos clientes estarão cientes de que existe um problema.
