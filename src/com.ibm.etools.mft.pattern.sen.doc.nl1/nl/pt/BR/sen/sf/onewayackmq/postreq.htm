Tarefas a serem concluídas depois de gerar o padrão de Service Facade para WebSphere MQ: unidirecional com reconhecimento

As tarefas que você deve concluir após ter gerado o padrão.

  1. Revise o CCSID. Este padrão assume que o CCSID usado para pedidos SOAP é 1208. se o aplicativo do provedor requerer mensagens com um CCSID diferente, este parâmetro deverá ser configurado no subfluxo RequestProcessing usando o editor de Fluxo de Mensagens. Esta propriedade não pode ser modificada com o editor de Broker Archive.
  2. Revise a configuração do nó SOAP. No caso de WSDLs complexos, pode ser necessário selecionar o WSDL a partir do editor de Fluxo de Mensagens, este requisito é mostrado no arquivo de resumo.
  3. Crie o arquivo broker archive (BAR). Inclua o fluxo de mensagens gerado em seu arquivo BAR. No editor do Broker Archive, você deve selecionar a opção Compilar e Sequenciar Recursos antes de construir o arquivo BAR. Os conjuntos de mensagens que são referenciados pelo fluxo também devem ser inclusos.
  4. Configure o arquivo broker archive. A configuração adicional do fluxo de mensagens no arquivo broker archive não é necessária antes de implementar os fluxos de mensagens para sua instância padrão de Service Facade para WebSphere MQ: unidirecional com reconhecimento, mas convém revisar algumas das opções padrão e configurar mudanças para seu ambiente. Por exemplo, talvez você deseje revisar as propriedades a seguir:
    • LoggingOn. Se a criação de log estiver incluída, no momento da implementação este comutador determina se a criação de log de pedidos é necessária para o ambiente atual.
    • ErrorLoggingOn. Se mensagens de erro forem incluídas, no momento da implementação este comutador determina se a criação de log de erros é necessária para o ambiente atual.
    É possível alterar qualquer um dos parâmetros padrão que são listados nas tabelas de parâmetros como configuráveis, mas é necessário ser cuidadoso porque o padrão pode estar assumindo comportamento específico. Entretanto, é possível alterar as propriedades que são determinadas pelas características operacionais e não afetam a lógica do fluxo, por exemplo, a configuração de segurança ou instâncias adicionais.
  5. Crie filas. Quando aplicar o padrão, uma lista de filas do WebSphere MQ será gerada, com nomes de filas e nomes do gerenciador de filas, estas filas do WebSphere MQ devem ser criadas se ainda não existirem. Todas as configurações de mensagem que são necessárias para o padrão são configuradas explicitamente nas mensagens e não são dependentes das configurações de filas.
    • No arquivo de resumo, visualize a lista de filas que você deve criar. O arquivo de resumo está localizado na pasta Configuração de Padrão do projeto de instância padrão InstanceName.
    • Crie recursos do WebSphere MQ. É possível criar recursos do WebSphere MQ na linha de comandos usando programas e utilitários como runmqsc ou usando a interface gráfica do WebSphere MQ Explorer. Para obter informações adicionais sobre a criação de recursos do WebSphere MQ, consulte WebSphere MQ Intercommunication, disponível na página da Web da biblioteca do WebSphere MQ.
  6. Implemente o arquivo BAR. O conjunto de mensagens contendo o WSDL implementável deve ser implementado, usando o mesmo arquivo BAR ou outro, em cada broker no qual os fluxos são implementados.
