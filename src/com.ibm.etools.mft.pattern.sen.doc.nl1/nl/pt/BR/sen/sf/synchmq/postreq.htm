Tarefas para Concluir Após Gerar o Padrão de Service Facade para WebSphere MQ: Pedido-Resposta

As tarefas que você deve concluir após ter gerado o padrão.

  1. Crie o arquivo broker archive (BAR). O fluxo de mensagens deve ser incluído em um arquivo bar. Qualquer conjunto de mensagens que é referenciado pelo fluxo é incluído automaticamente.
    1. Clique em Arquivo > Novo > Arquivo BAR.
    2. Digite o nome do projeto do servidor ou selecione da lista exibida. A lista é filtrada para mostrar apenas projetos no conjunto de trabalhos ativo.
    3. Insira um nome para o arquivo BAR sendo criado.
    4. Clique em Concluir. É criado um arquivo com uma extensão .bar e exibido na visualização do Navegador de Administração do Intermediário, sob a pasta Broker Archives. O editor de Conteúdo para o arquivo BAR é aberto.
    5. Na guia Preparar, selecione o arquivo de fluxo de mensagens gerado para incluí-lo em seu arquivo BAR.
    6. Selecione Compilar e Sequenciar Recursos e, em seguida, clique em Construir e Salvar para criar seu arquivo BAR.
  2. Configure o arquivo broker archive. A configuração adicional do fluxo de mensagens no arquivo bar não é necessária antes de implementar os fluxos de mensagens para sua instância padrão de Service Facade para WebSphere MQ: Pedido-Resposta, mas talvez você deseje revisar algumas das opções padrão e configurar alterações para seu ambiente. Na página Gerenciar do editor de Broker Archive:
    1. Opcional: Use a propriedade definida pelo usuário RequestLoggingOn para selecionar se a criação de log está ativada ou desativada para o fluxo de mensagens de Solicitação.
    2. Opcional: Use a propriedade definida pelo usuário ResponseLoggingOn para selecionar se a criação de log está ativada ou desativada para o fluxo de mensagens de Resposta.
    3. Opcional: Use a propriedade definida pelo usuário ErrorLoggingOn para selecionar se as mensagens de erro são geradas.
    4. Opcional: É possível alterar qualquer uma das propriedades configuráveis no broker archive, por exemplo, propriedades para segurança ou criar instâncias adicionais.

      É necessário assegurar que quaisquer valores que você alterar não causem impacto no comportamento da instância padrão.

  3. Crie filas. Quando aplicar o padrão, uma lista de filas do WebSphere MQ será gerada com nomes de filas e nomes de gerenciador de filas, estas filas do WebSphere MQ deverão ser criadas se ainda não existirem. Todas as configurações de mensagem que são necessárias para o padrão são configuradas explicitamente nas mensagens e não são dependentes das configurações de filas.
    1. No arquivo de resumo, visualize a lista de filas que você deve criar. O arquivo de resumo está localizado na pasta Configuração de Padrão do projeto de instância padrão InstanceName.
    2. Crie recursos do WebSphere MQ. É possível criar recursos do WebSphere MQ na linha de comandos usando programas e utilitários como runmqsc ou usando a interface gráfica do WebSphere MQ Explorer. Para obter informações adicionais sobre a criação de recursos do WebSphere MQ, consulte WebSphere MQ Intercommunication, disponível na página da Web da biblioteca do WebSphere MQ.
  4. Implemente o arquivo BAR. O conjunto de mensagens contendo o WSDL implementável deve ser implementado, usando o mesmo arquivo BAR ou outro, em cada broker no qual os fluxos são implementados.
