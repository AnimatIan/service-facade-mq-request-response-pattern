Subfluxo de Log para o Padrão de Service Facade para WebSphere MQ: Pedido-Resposta

As mensagens de criação de log para um banco de dados não ocorrem nos fluxos de mensagens que manipulam transações de negócios. Em vez disso, mensagens persistentes são gravadas em uma fila do WebSphere MQ, que é processada separadamente. Gravar mensagens persistentes em uma fila do WebSphere MQ isola qualquer impacto no desempenho de atualizações de banco de dados, permitindo, portanto, o controle de onde e quando estas filas são processadas.

O padrão assume que os dados do log são incluídos no cabeçalho MQRFH2 se um existir. Se nenhum cabeçalho MQRFH2 existir, um será criado. O corpo da mensagem é registrado conforme ele chega no fluxo de mensagens. Os dados do log a seguir são incluídos:

  • Registro de data e hora do log
  • Nome do broker
  • Nome do fluxo de mensagens
  • Status (0 sucesso)
  • Modo - Pedido / Resposta
