Parâmetros para o Padrão de Service Facade para WebSphere MQ: Pedido-Resposta

Informações de Serviço
Parâmetro de padrão M C Default Descrição
WSDL de Serviço sim Não Não se aplica Este parâmetro padrão define o arquivo WSDL implementável que define a interface de serviço apresentada pelo broker aos solicitantes.

Este valor deve incluir o nome do caminho que se inicia no projeto do conjunto de mensagens da área de trabalho.

Para navegar até o WSDL requerido, clique em Navegar.
Validação do Pedido SOAP Não sim Nenhum Pedidos de serviço recebidos são validados conforme definido por este parâmetro. Os valores válidos são:
  • Nenhum
  • Conteúdo e Valor
Validação da resposta SOAP Não sim Nenhum Respostas SOAP de saída são validadas conforme definido por este parâmetro. Os valores válidos são:
  • Nenhum
  • Conteúdo e Valor
Informações de resposta
Parâmetro de padrão M C Default Descrição
Fila de Respostas sim sim RESPONSE Este parâmetro padrão identifica a fila do broker para a qual o aplicativo do provedor deve retornar respostas.
Fila de armazenamento sim sim STORE Este parâmetro padrão define a fila que é usada para armazenar os cabeçalhos MQMD enquanto aguarda uma resposta do provedor.
Expiração das mensagens de fila de armazenamento em décimos de um segundo sim sim Sem tempo limite Este parâmetro padrão define o tempo de expiração, em décimos de um segundo, para mensagens que contêm o identificador de resposta HTTP.

Este valor deve exceder o tempo de resposta máximo esperado.

É possível inserir seu próprio valor ou aceitar o valor-padrão Sem tempo limite.
Informações do Provedor
Parâmetro de padrão M C Default Descrição
Gerenciador de filas do provedor Não sim   Este parâmetro padrão identifica o gerenciador de filas no qual o aplicativo do provedor recebe pedidos do fluxo de mensagens do broker.

Se for deixado em branco, o padrão é o gerenciador de filas do broker.
Fila de pedidos do provedor sim sim PROVIDER Este parâmetro padrão identifica a fila na qual o aplicativo do provedor recebe pedidos do fluxo de mensagens do broker.
Criação de log
Parâmetro de padrão M C Default Descrição
Criação de log requerida Não Não Limpo

Este parâmetro padrão determina se a instância padrão inclui o código para criação de log.

Se Criação de log requerida estiver limpo, o subfluxo de Log não será incluído.

Se Criação de log requerida estiver selecionado, um subfluxo de Log será incluído no fluxo de mensagens e os valores deverão ser configurados para Gerenciador de filas de log e Fila de logs.
Gerenciador de filas de log Não Sim

Este parâmetro padrão define o gerenciador de filas para criação de log. Ele poderá ser deixado em branco se o gerenciador de filas do broker for usado para criação de log.

Este parâmetro é necessário apenas se Criação de log requerida é selecionado.

Fila de Logs Não Sim LOG

Este parâmetro padrão define a fila para criação de log.

Este parâmetro é necessário apenas se Criação de log requerida é selecionado.
Tratamento de erro
Parâmetro de padrão M C Default Descrição
Mensagem de erro necessária Não Não Selecionado

Este parâmetro padrão define se o aplicativo padrão cria os elementos de fluxo de mensagens para criar mensagens de erro.

Se Mensagem de erro requerida for selecionado, os valores deverão ser configurados para Gerenciador de filas de erros e Fila de erros.
Gerenciador de filas de erros Não Sim  

Este parâmetro padrão define o gerenciador de filas para mensagens de erro. Ele poderá ser deixado em branco se o gerenciador de filas do broker for usado para criação de log.
Fila de erros Não Sim ERROR

Este parâmetro padrão define a fila para mensagens de erro.

Este parâmetro é necessário apenas se Mensagem de erro requerida é selecionado.

Gerais
Parâmetro de padrão M C Default Descrição
Esquema do Servidor Intermediário Não Não mqsi Um esquema do broker pode ser usado para criar um espaço de nomes para a instance padrão.

Se configurado, todos os fluxos de mensagens e arquivos ESQL são criados no esquema do broker denominado. Se não configurado, o esquema padrão será usado.

Prefixo do fluxo Não Não   Se configurado, este prefixo é incluído no início de todos os nomes de fluxo de mensagens.
Sufixo do fluxo Não Não   Se configurado, este sufixo é incluído no final de todos os nomes de fluxo de mensagens.
Prefixo de Fila Não Não   Se configurado, este prefixo é incluído no início de todos os nomes da fila.

Se os nomes de fila padrão forem usados, o prefixo, o sufixo, ou ambos, deverão ser configurados para identificar nomes de fila para cada instância padrão.

Sufixo da Fila Não Não   Se configurado, este sufixo é incluído no final de todos os nomes de fila.
Descrição curta Não Não   Esta descrição é incluída na propriedade Descrição Resumida de todos os fluxos de mensagens que são gerados.
Descrição detalhada Não Não   Esta descrição é incluída na propriedade Descrição Detalhada de todos os fluxos de mensagens que são gerados.

Todos os parâmetros de padrão são listados com seus valores-padrão.

A coluna intitulada O indica se a propriedade é obrigatória. A coluna intitulada C indica se a propriedade é configurável, significando que é possível alterar o valor quando você inclui o fluxo de mensagens no arquivo broker archive (BAR).

Onde nenhum padrão é indicado, o valor-padrão é uma cadeia vazia.
