Tarefas para Concluir Antes de Aplicar o Padrão de Service Facade para WebSphere MQ: Pedido-Resposta

Antes de aplicar o padrão para produzir uma instância padrão, é necessário determinar quais das tarefas a seguir são necessárias. Se necessário, conclua as tarefas a seguir:

  • Identifique se este padrão atende seu requisito de negócios e pode ser suportado pelos aplicativos de solicitante e do provedor.
  • Defina ou identifique a interface do provedor e a interface de serviço associada.
  • Nomeie a instância padrão. Identifique o propósito do aplicativo padrão para que você possa decidir sobre um nome apropriado.
  • Selecione as filas e gerenciadores de filas do WebSphere MQ. Os gerenciadores de filas e filas existentes podem ser usados, por exemplo, para acessar o aplicativo do provedor e é necessário escolher seus nomes.
  • Identifique quaisquer gerenciadores de filas e filas adicionais que sejam necessários para este padrão e decida sobre seus nomes. Leve em conta quaisquer políticas, normas ou convenções de nomenclatura que devem ser aplicadas.
  • Decida se você requer a criação de log. Se a criação de log for necessária, é preciso decidir para qual gerenciador de filas e fila as mensagens de criação de log são enviadas. É possível enviar mensagens de criação de log para uma fila de log geral que é usada por múltiplos fluxos de mensagens ou você pode usar uma fila especificamente para a instância padrão.
  • Decida se você requer mensagens de erro.
  • Decida se você requer validação.
  • Crie o WSDL implementável em um projeto do conjunto de mensagens.
