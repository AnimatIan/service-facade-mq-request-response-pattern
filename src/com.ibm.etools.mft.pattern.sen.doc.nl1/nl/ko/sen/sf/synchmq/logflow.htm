WebSphere MQ에 서비스 퍼사드: 요청-응답 패턴의 로그 서브플로우

비즈니스 트랜잭션을 처리하는 메시지 플로우에서 데이터베이스에 메시지를 로그하지 않습니다. 대신, 따로 처리되는 지속 메시지가 WebSphere MQ 큐에 기록됩니다. 지속 메시지를 WebSphere MQ 큐에 기록하면 데이터베이스 업데이트의 성능 영향이 격리되므로 이 큐가 처리되는 위치와 시기를 제어할 수 있습니다.

해당 패턴은 로그 데이터가 MQRFH2 헤더에 추가된다고 가정합니다(존재하는 경우). MQRFH2 헤더가 존재하지 않으면 MQRFH2 헤더가 작성됩니다. 메시지 본문은 메시지 플로우에 도착할 때 기록됩니다. 다음과 같은 로그 데이터가 추가됩니다.

  • 로그 시간소인
  • 브로커 이름
  • 메시지 플로우 이름
  • 상태( 0 성공)
  • 모드 - 요청 / 응답
