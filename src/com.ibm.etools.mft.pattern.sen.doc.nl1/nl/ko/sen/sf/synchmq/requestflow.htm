WebSphere MQ에 서비스 퍼사드: 요청-응답 패턴의 요청 플로우

WebSphere MQ에 서비스 퍼사드: 요청-응답 패턴을 위한 로그 기록 및 예외를 포함한 요청 플로우 세부사항입니다.

  1. SOAP 요청은 SOAP 노드에서 수신합니다. 각 패턴 인스턴스의 패턴 매개변수가 유효성 검증을 필요로 하는 경우 요청을 유효성 검증합니다.
  2. 제공자 애플리케이션에서 기본 XML 메시지를 대상으로 유지하도록 SOAP 및 HTTP 헤더가 제거됩니다.
  3. 그러면 메시지가 패턴 인스턴스를 사용자 정의할 수 있는 지점을 제공하는 요청 처리 서브플로우에 전달됩니다.
  4. 메시지 브로커의 응답 주소에 MQMD 헤더를 삽입하고 예상 응답 시간에 맞게 만기 시간이 설정됩니다.
  5. 메시지가 제공자 큐에 배치됩니다.
  6. SOAP 응답 ID를 저장할 메시지가 설정됩니다. 보내는 메시지에 대해 생성된 메시지 ID는 로컬 환경의 기록된 목적지 데이터에서 검색하고 MQMD 상관 ID에 사용됩니다. SOAP 응답 ID는 로컬 환경에서 검색하고 메시지의 본문으로 사용됩니다. 이 데이터는 내부 큐에 기록됩니다.
  7. 메시지를 제공자로 송신하기 전에 예외가 발생하면 SOAP 결함이 제공자로 리턴됩니다. 오류 메시지가 필요하면 오류 메시지를 오류 큐로 송신합니다.
  8. 요청 로그 기록이 필요하고 메시지 플로우가 성공하는 경우, 수신되는 메시지 로그를 기록하도록 로그 기록 서브플로우가 호출됩니다.
