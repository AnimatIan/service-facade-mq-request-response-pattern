WebSphere MQ에 서비스 퍼사드: 요청-응답 패턴의 응답 플로우

응답 플로우는 비트랜잭션 플로우로 실행됩니다.

제공자 애플리케이션으로부터 응답을 수신하면 다음과 같은 조치가 수행됩니다.

  1. 응답 플로우는 제공자의 응답 메시지를 읽습니다. 패턴 매개변수에 따라 유효성 검증이 결정됩니다.
  2. 응답 플로우는 상관 ID를 사용하여 내부 WebSphere MQ 큐에서 SOAP 응답 ID를 검색합니다. SOAP 응답 ID가 로컬 환경에 추가되어 SOAP 응답 대상을 제어합니다.
  3. HTTP 응답 헤더가 작성됩니다.
  4. 예외가 발생하고 패턴 애플리케이션에 오류 로그 기록이 포함되어 있으면 오류 서브플로우가 호출되고 오류가 응답 큐에 기록됩니다. 하지만, 실패는 SOAP 요청에 응답하는 데 필요한 ReplyID를 사용할 수 없음을 의미하기 때문에 요청자에 대해 응답이 시도되지 않습니다.
  5. 성공 응답은 로그 기록이 이 패턴 애플리케이션에 포함되어 있고 현재 설정된 상태인 경우에만 기록됩니다.
