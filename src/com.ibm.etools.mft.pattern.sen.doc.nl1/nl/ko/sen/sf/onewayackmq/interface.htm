인터페이스 예

서비스 인터페이스로 사용되는 WSDL 파일의 요청 요소에 대한 기반으로 제공자 애플리케이션에서 예상한 XML 메시지의 스키마가 필요합니다.

예를 들어, 제공자 애플리케이션은 조치를 포함한 메시지를 가져올 수 있으며, 해당 메시지는 조치를 수행해야 하는 항목을 포함하고 있습니다. 다음 예에는 서비스 조작이 되는 SaveItem 조치를 설명합니다. 

	<?xml version="1.0" encoding="UTF-8"?>
	<schema targetNamespace="http://ServiceFacade_RR" elementFormDefault="qualified"
	xmlns="http://www.w3.org/2001/XMLSchema" xmlns:tns="http://ServiceFacade_RR">
	<complexType name="항목">
	<sequence>
	<element name="이름" type="string"></element>
	<element name="코드" type="string"></element>
	<element name="설명" type="string"></element>
	<element name="위치" type="string"></element>
	</sequence>
	</complexType>
	<element name="SaveItem" type="tns:SaveItem"></element>
	<complexType name="SaveItem">
	<sequence>
	<element name="항목" type="tns:Item"></element>
	</sequence>
	</complexType>
	</schema>

연관된 WSDL가 제공자 스키마에 필요한 최상위 레벨 요소에 상응하는 요소로 입력 메시지를 정의해야 합니다. 다음 부분이 선행 제공자 스키마에 해당합니다. 

	<wsdl:message name="SaveItemRequestMsg"> <wsdl:part name="SaveItemRequest" 
	element="tns:SaveItem"></wsdl:part> </wsdl:message>

수신확인

수신확인은 패턴에 따라 고정되고 모든 패턴 인스턴스에 적용되지만, 네임스페이스는 다양합니다.

수신확인의 스키마(패턴 고유의 네임스페이스 표시)는 다음과 같습니다. 

	<?xml version="1.0" encoding="UTF-8"?> 
	<schema targetNamespace="<instance specific>" 
	elementFormDefault="qualified" xmlns="http://www.w3.org/2001/XMLSchema" 
	xmlns:tns="http://<instance specific>" "> 
	<complexType name="Ack"> 
	<sequence> 
	<element name="acknowledgment" type="string"></element> 
	</sequence> 
	</complexType> 
	<element name="Response" type="tns:Ack">
	</element> 
	</schema>

응답 메시지를 정의하는 WSDL 항목이 다음 응답을 참조해야 합니다. 

	<wsdl:message name="SaveItemResponseMsg"> <wsdl:part name="응답"
	element="tns:Response"></wsdl:part> </wsdl:message>
