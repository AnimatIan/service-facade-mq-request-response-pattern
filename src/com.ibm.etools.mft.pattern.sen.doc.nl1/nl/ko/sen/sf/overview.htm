서비스 퍼사드 패턴

서비스 퍼사드 패턴을 사용하면 애플리케이션에서 제공하는 기능 또는 서비스가 실행되지 않는 기능과 서비스 요청자 간의 완화된 커플링을 제공할 수 있습니다. 느슨한 결합은 제공자의 복잡도를 보이지 않고 표준 웹 서비스 인터페이스를 제공합니다.

서비스 퍼사드 패턴은 로그 기록과 같은 표준 기능 애플리케이션에 대한 중개 지점도 도입하고 있습니다.

컨텍스트

엔터프라이즈 아키텍처가 서비스 지향 아키텍처로 전환할 때는 비즈니스의 많은 부분을 실행하지만 보안 기능이 없는 기존의 레거시 시스템을 포용해야 하는 경우가 많습니다. 이 애플리케이션은 대개 최신 서비스 실행 소프트웨어 패키지에서 액세스해야 하지만 어댑터, 메시징 또는 기타 서비스 지향이 아닌 통합 기술을 통한 액세스만 제공하는 기능을 사용합니다.

문제점

레거시 애플리케이션은 기본적으로 서비스 지향 아키텍처(SOA) 환경에 적합하지 않으며 이러한 시스템을 변경하는 것은 비용이 많이 들고 특수 기술을 필요로 합니다. 문제점은 최신 서비스 지향 패키지 및 애플리케이션과 이러한 시스템을 통합하는 방법을 찾는 것입니다. 특히, 일반적으로 웹 서비스와 함께 사용하는 동기 HTTP 프로토콜과 레거시 애플리케이션이 자주 사용하는 메시징 프로토콜 간 브릿지를 작성해야 합니다.

자세한 내용

이 패턴은 엔터프라이즈가 SOA 환경의 부분으로 요청 클라이언트에 서비스 인터페이스를 제공하려고 하지만 서비스 인터페이스 제공을 위해 제공자 애플리케이션을 쉽게 업그레이드할 수 없는 경우 적합합니다.

이 패턴은 제공자 애플리케이션이 XML 인터페이스를 제공하고 클라이언트 애플리케이션이 웹 서비스 호출을 지원하는 경우에 사용됩니다. 변환으로 패턴을 확장하여 WebSphere MQ를 통해 비XML 인터페이스로 애플리케이션에 대한 서비스 퍼사드를 지원할 수 있습니다.
