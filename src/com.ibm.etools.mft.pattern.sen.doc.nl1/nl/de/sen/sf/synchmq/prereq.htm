Tasks, die vor der Anwendung des Musters Service Facade to WebSphere MQ: request-response (Servicefassade für WebSphere MQ: Anforderung-Antwort) ausgeführt werden müssen

Bevor Sie das Muster für die Erstellung einer Musterinstanz anwenden, müssen Sie bestimmen, welche der folgenden Tasks erforderlich sind. Führen Sie folgende Tasks aus, falls erforderlich:

  • Prüfen Sie, ob dieses Muster Ihren Geschäftsanforderungen entspricht und sowohl von den Requester- als auch von den Provideranwendungen unterstützt wird.
  • Definieren bzw. ermitteln Sie die Providerschnittstelle sowie die zugehörige Serviceschnittstelle.
  • Wählen Sie einen Namen für die Musterinstanz aus. Legen Sie einen aussagekräftigen Namen fest, der auf den Zweck der Musteranwendung hinweist.
  • Wählen Sie die WebSphere MQ-Warteschlangen und -Warteschlangenmanager aus. Vorhandene Warteschlangenmanager und Warteschlangen können zum Beispiel für den Zugriff auf die Provideranwendung verwendet werden, und deren Namen müssen festgelegt werden.
  • Ermitteln Sie alle weiteren für dieses Muster erforderlichen Warteschlangenmanager und Warteschlangen und legen Sie deren Namen fest. Berücksichtigen Sie dabei alle anzuwendenden Regeln, Standards und Namenskonventionen.
  • Entscheiden Sie, ob die Protokollierung erforderlich ist. Falls ja, bestimmen Sie den Warteschlangenmanager und die Warteschlange für die Protokollnachrichten. Die Protokollnachrichten können an eine allgemeine Protokollwarteschlange gesendet werden, die von mehreren Nachrichtenflüssen verwendet wird, oder an eine speziell für die Musterinstanz eingerichtete Warteschlange.
  • Entscheiden Sie, ob Fehlernachrichten erforderlich sind.
  • Entscheiden Sie, ob die Validierung erforderlich ist.
  • Erstellen Sie die implementierbare WSDL in einem Nachrichtensatzprojekt.
