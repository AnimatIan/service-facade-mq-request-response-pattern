Vorgaben für die Verwendung des Musters Service Facade to WebSphere MQ: one-way with acknowledgment (Servicefassade für WebSphere MQ: unidirektional mit Empfangsbestätigung)

Wenn Sie dieses Muster verwenden, gelten eine Reihe von Vorgaben.

Sie können das Muster Service Facade to WebSphere MQ: one-way with acknowledgment (Servicefassade für WebSphere MQ: unidirektional mit Empfangsbestätigung) nur verwenden, wenn alle der folgenden Vorgaben erfüllt sind:

  • Die Anforderungen werden als SOAP-Nachrichten über HTTP im literalen Dokumentstil empfangen.
  • SOAP-Anforderungen und -Antworten verwenden CCSID 1208.
  • Die anfordernden Anwendungen unterstützen das Muster Service Facade to WebSphere MQ: one-way with acknowledgment (Servicefassade für WebSphere MQ: unidirektional mit Empfangsbestätigung).
  • Sobald eine Nachricht an ihre Eingabewarteschlange gesendet wurde, sind die Provideranwendungen für die Aktionen bei einer Aktualisierung zuständig.
  • Die WSDLs, die die Brokerschnittstelle definieren, weisen nur einen Porttyp und nur eine Bindung auf.
  • Die WSDLs müssen die Eigenschaften für den SOAPInput-Knoten definieren, der in der WSDL der höchsten Ebene definiert ist. Einige komplexe, verschachtelte WSDLs können daher nicht automatisch von diesem Muster verarbeitet werden.

    Wenn die verwendeten WSDLs diese Bedingungen nicht erfüllen, kann das Muster den SOAPInput-Knoten nicht konfigurieren. Die Konfiguration des SOAPInput-Knotens wird in diesem Fall als erforderliche Task nach der Generierung der Musterinstanz aufgeführt.

  • Die Provideranwendung unterstützt WebSphere MQ-XML-Nachrichten in einem Format, das als Hauptteil der SOAP-Anforderung verwendet werden kann. Sie können den untergeordneten Anforderungsprozessor-Nachrichtenfluss aber auch so anpassen, dass das Format für Nicht-XML-Nachrichten umgesetzt wird.
  • Das Muster definiert sein eigenes Format für Protokoll- und Fehlernachrichten. Es wird davon ausgegangen, dass diese Nachrichten den Anforderungen in Ihrer Umgebung entsprechen.

Einige dieser Einschränkungen, insbesondere die CCSID, lassen sich durch eine entsprechende Konfiguration nach der Anwendung des Musters umgehen.
