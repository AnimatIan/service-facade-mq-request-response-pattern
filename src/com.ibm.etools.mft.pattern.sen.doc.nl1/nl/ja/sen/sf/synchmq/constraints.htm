「WebSphere MQ へのサービス・ファサード: 要求/応答」パターンの使用に関する制約

このパターンを使用する場合に適用される制約は多数あります。

以下のすべての制約を満たす場合にのみ、「WebSphere MQ へのサービス・ファサード: 要求/応答」パターンを使用できます。

  • HTTP を介した SOAP メッセージとして要求を受け取る必要がある。
  • 要求元アプリケーションが同期要求応答対話スタイルをサポートしている必要がある。
  • プロバイダー・アプリケーションが、同期対話に適した時間フレームで要求に応答できる必要がある。
  • プロバイダー・アプリケーションが WebSphere MQ 経由で要求を受け取り、応答を返すことができる。
  • プロバイダー・アプリケーションが、メッセージ ID を相関 ID にコピーするという標準の動作をサポートしている必要がある。
  • ブローカー・インターフェースを定義する WSDL が 1 つのポート・タイプおよび 1 つのバインディングのみを有している。
  • WSDL には、最上位の WSDL の定義に、SOAPInput ノードのプロパティーを定義する定義が含まれている必要があります。 このため、このパターンでは、ネストされた一部の複雑な WSDL は自動的に処理できません。

    この制約を満たさない WSDL が使用されている場合、パターンは SOAPInput ノードを構成しませんが、パターン・インスタンスの生成後に完了するタスクとして SOAPInput ノードの構成をリストします。

  • プロバイダー・アプリケーションは、SOAP 要求および応答の本体として使用可能な形式で WebSphere MQ XML メッセージをサポートします。あるいは、要求処理サブフローをカスタマイズして、形式を非 XML メッセージに変換することもできます。
  • ロギングが必要な場合、このパターンで使用されるフォーマットが要件を満たしていると想定されます。
