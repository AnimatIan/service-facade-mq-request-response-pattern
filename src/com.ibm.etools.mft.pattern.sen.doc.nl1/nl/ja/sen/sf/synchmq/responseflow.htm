「WebSphere MQ へのサービス・ファサード: 要求/応答」パターンの応答フロー

応答フローは非トランザクション・フローとして実行されます。

プロバイダー・アプリケーションから応答が受信されると、以下のアクションが実行されます。

  1. 応答フローは、プロバイダーからの応答メッセージを読み取ります。 妥当性検査は、パターン・パラメーターによって判別されます。
  2. フローは相関 ID を使用して SOAP 応答 ID を内部 WebSphere MQ キューから取り出します。 SOAP 応答 ID は、SOAP 応答の宛先を制御するためにローカル環境に追加されます。
  3. HTTP 応答ヘッダーが作成されます。
  4. 例外が発生した場合、エラー・ロギングがパターン・アプリケーションに組み込まれていれば、エラー・サブフローが呼び出されて、エラーが応答キューに書き込まれます。 ただし、障害は SOAP 要求に応答する必要のあった ReplyID が使用不可であることを暗黙に示しているので、要求元に対する応答は試行されません。
  5. 正常な応答は、ロギングがこのパターン・アプリケーションに組み込まれていて現在オンになっている場合にのみ、ログに記録されます。
