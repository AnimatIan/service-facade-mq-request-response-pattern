「WebSphere MQ へのサービス・ファサード: 確認応答を伴う一方向」パターンの使用に関する制約

このパターンを使用する場合に適用される制約は多数あります。

以下のすべての制約を満たす場合にのみ、「WebSphere MQ へのサービス・ファサード: 確認応答を伴う一方向」パターンを使用できます。

  • 要求が、文書リテラル・スタイルを使用する、HTTP を介した SOAP メッセージとして受け取られる。
  • SOAP 要求および応答が CCSID 1208 を使用する。
  • 要求アプリケーションが「WebSphere MQ へのサービス・ファサード: 確認応答を伴う一方向」パターンをサポートできる。
  • プロバイダー・アプリケーションが、メッセージが入力キューに送信されるときの更新に従って動作する責任を負っている。
  • ブローカー・インターフェースを定義する WSDL が 1 つのポート・タイプおよび 1 つのバインディングのみを有している。
  • WSDL には、最上位の WSDL の定義に、SOAPInput ノードのプロパティーを定義する定義が含まれている必要があります。 このため、このパターンでは、ネストされた一部の複雑な WSDL は自動的に処理できません。

    この制約を満たさない WSDL が使用されている場合、パターンは SOAPInput ノードを構成しませんが、パターン・インスタンスの生成後に完了するタスクとして SOAPInput ノードの構成をリストします。

  • プロバイダー・アプリケーションは、SOAP 要求の本体として使用可能な形式で WebSphere MQ XML メッセージをサポートします。あるいは、要求処理サブフローをカスタマイズして、形式を非 XML メッセージに変換することもできます。
  • パターンはロギングおよびエラー・メッセージ用の独自のフォーマットを定義しています。これらのメッセージは実際の環境での要件を十分に満たすと想定されます。

前述の制約の一部 (特に CCSID) は、パターンが適用された後に構成することによって回避できます。
