Service Facade to WebSphere MQ: one-way with acknowledgment 模式

使用 Service Facade to WebSphere MQ: one-way with acknowledgment 模式将 Web Service 接口呈现给客户机，并使用支持 WebSphere MQ 的应用程序来实现服务请求。

使用此模式来搭建异步 HTTP 协议与可靠消息传递协议之间的网桥以处理更新，并确保已保存请求以供处理。

此模式在请求者和提供者之间提供了计时、协议和传输方面的松耦合。它适合于现有系统的服务接口。

解决方案

解决方案用于实现提供服务入口点的消息流。通过 HTTP 接收 Web Service 请求时，数据会作为持久消息放置在提供者应用程序的队列上。放置消息时，将返回确认，以确认请求被接受。如果客户机收到确认，那么表示消息已写入队列中。如果收到故障，那么表示没有发送任何消息；不过如果是超时故障，那么情况不确定，但是客户机应用程序发现了存在问题。
