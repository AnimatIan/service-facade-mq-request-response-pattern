Service Facade 模式

使用 Service Facade 模式在服务请求者与不支持服务的应用程序或功能所提供的功能之间提供松耦合。松耦合会隐藏提供者的复杂性并提供标准的 Web Service 接口。

Service Facade 模式也为标准功能（如记录）的应用程序引入了调解点。

上下文

当企业体系结构转变为面向服务的体系结构时，往往必须承载运行大部分业务却没有服务功能的现有旧系统。这些应用程序通常使用的功能都必须让支持服务的较新软件包能够访问，但它们却只能提供通过适配器、消息传递或其他非面向服务的集成技术的访问。

问题

旧应用程序并不会自然而然地与面向服务的体系结构 (SOA) 环境相适应，而对此类系统进行更改不但价格高昂，而且技术稀缺。问题在于找到一种方法使此类系统与面向服务的较新软件包和应用程序相集成。特别是，必须在通常供 Web Service 使用的同步 HTTP 协议和通常供旧应用程序使用的消息传递协议之间创建网桥。

选择指导信息

如果企业希望将服务接口作为 SOA 环境的一部分提供给请求客户机，而提供者应用程序无法轻松升级以提供服务接口，那么就适合使用此模式。

此模式的使用情况是：提供者应用程序提供 XML 接口且客户机应用程序支持调用 Web Service。此模式可通过变换进行扩展，以支持在 WebSphere MQ 上使用非 XML 接口应用程序的服务外观。
