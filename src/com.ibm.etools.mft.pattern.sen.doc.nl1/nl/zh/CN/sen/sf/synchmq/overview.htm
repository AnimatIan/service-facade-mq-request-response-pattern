Service Facade to WebSphere MQ: request-response 模式

使用 Service Facade to WebSphere MQ: request-response 模式向只能通过 WebSphere MQ 访问的功能提供 Web Service 外观。此模式在通常供 Web Service 使用的同步 HTTP 协议和无法轻松升级的具有 WebSphere MQ 接口的现有应用程序之间创建网桥。

使用此模式的情况是：提供者应用程序提供 XML 接口且客户机应用程序支持调用 Web Service。此模式可通过变换进行扩展，以支持在 WebSphere MQ 上使用非 XML 接口应用程序的服务外观。

解决方案

解决方案用于实现提供服务入口点的消息流。收到服务请求时，会将应答标识存储在内部队列上并将请求转发到提供者应用程序。从提供者应用程序收到响应时，将恢复应答标识并将包含提供者响应的 Web Service 应答返回到请求应用程序。
