使用 Service Facade to WebSphere MQ: request-response 模式时的约束

使用此模式时会有若干约束。

只有满足以下所有约束时，才可以使用 Service Facade to WebSphere MQ: request-response 模式：

  • 请求的接收形式必须是 HTTP 上的 SOAP 消息。
  • 请求应用程序必须支持同步请求响应交互样式。
  • 提供者应用程序必须能够在适合同步交互的时间范围内对请求作出响应。
  • 提供者应用程序可以通过 WebSphere MQ 接收请求并返回响应。
  • 提供者应用程序必须支持将消息标识复制到相关标识的标准行为。
  • 定义代理接口的 WSDL 只有一个端口类型和一个绑定。
  • WSDL 的顶级 WSDL 定义中必须包含定义 SOAPInput 节点属性的定义。因此，该模式无法自动处理某些复杂嵌套的 WSDL。

    如果使用了不符合此约束的 WSDL，那么该模式不会配置 SOAPInput 节点，而是会将配置 SOAPInput 节点列为生成模式实例之后要完成的任务。

  • 提供者应用程序支持的 WebSphere MQ XML 消息格式是可用作 SOAP 请求和响应主体的格式，您也可以定制请求处理子流以将格式变换为非 XML 消息。
  • 如果需要记录，那么会假定此模式使用的格式符合您的要求。
