Service Facade to WebSphere MQ: request-response 型樣的回應流程

「回應」流程是以非交易式流程的形式執行。

接收到來自提供者應用程式的回應時，會採取下列動作：

  1. 「回應」流程會讀取提供者中的回應訊息。驗證是透過型樣參數來決定。
  2. 此流程會使用相關性 ID 以擷取內部 WebSphere MQ 佇列中的「SOAP 回覆 ID」。 「SOAP 回覆 ID」會新增至區域環境，以控制 SOAP 回覆的目的地。
  3. 會建立 HTTP 回覆標頭。
  4. 如果發生異常狀況，並將錯誤記載併入型樣應用程式中，則會呼叫「錯誤」子流程，並將錯誤寫入回應佇列中。 不過，因為失敗表示無法使用必須回覆 SOAP 要求的 ReplyID，所以不會嘗試回覆要求者。
  5. 只有在將記載併入此型樣應用程式中而且目前已開啟該功能時，才會記載順利完成的回應。
