Service Facade to WebSphere MQ: request-response 型樣

使用Service Facade to WebSphere MQ: request-response 型樣，可以將 Web 服務 Facade 提供給只能透過 WebSphere MQ 存取的功能。此型樣可在下列兩者之間建立橋接器：一般與 Web 服務搭配使用的同步 HTTP 通訊協定，與具有 WebSphere MQ 介面且無法輕易升級的現有應用程式。

如果提供者應用程式提供 XML 介面，而用戶端應用程式支援對 Web 服務的呼叫，則請使用此型樣。此型樣可以使用轉換加以延伸，以透過 WebSphere MQ 支援具有非 XML 介面的應用程式的服務 Facade。

解決方案

解決方案是實作可提供服務進入點的訊息流程。接收到服務要求時，回覆 ID 會儲存在內部佇列中，並將要求轉遞至提供者應用程式。接收到來自提供者應用程式的回應時，會回復回覆 ID，並將 Web 服務回覆（包含提供者回應）傳回給要求端應用程式。
