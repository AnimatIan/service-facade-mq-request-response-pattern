使用 Service Facade to WebSphere MQ: request-response 型樣的限制

當您使用此型樣時，將適用一些限制。

只有在符合下列所有限制時，才能使用 Service Facade to WebSphere MQ: request-response 型樣：

  • 要求必須透過 HTTP 以 SOAP 訊息形式接收。
  • 要求端應用程式必須支援同步要求回應互動樣式。
  • 提供者應用程式必須可以在同步互動適用的時間範圍內回應要求。
  • 提供者應用程式可以接收要求，並透過 WebSphere MQ 傳回回應。
  • 提供者應用程式必須支援將訊息 ID 複製至相關性 ID 的標準行為。
  • 定義分配管理系統介面的 WSDL 只有一個埠類型及一個連結。
  • WSDL 必須包含的定義會定義最上層 WSDL 定義中 SOAPInput 節點的內容。因此，型樣無法自動處理部分複式巢狀 WSDL。

    如果使用不符合此限制的 WSDL，則型樣不會配置 SOAPInput 節點，但會將 SOAPInput 節點的配置列為在產生型樣實例之後要完成的作業。

  • 提供者應用程式支援其格式可以作為 SOAP 要求及回應主體使用的 WebSphere MQ XML 訊息，或者，您可以自訂要求處理子流程，以將格式轉換為非 XML 訊息。
  • 如果需要記載，則會採用此型樣所使用的格式，以符合需求。
