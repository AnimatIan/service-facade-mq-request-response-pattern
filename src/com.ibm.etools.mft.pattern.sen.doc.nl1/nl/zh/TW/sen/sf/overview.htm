Service Facade 型樣

使用 Service Facade 型樣，可以在服務要求者與應用程式所提供的功能或未啟用服務的功能之間，提供鬆散聯結。 鬆散聯結可隱藏提供者的複雜性，並呈現標準 Web 服務介面。

Service Facade 型樣也會引進應用標準功能（例如記載）的調解點。

環境定義

企業架構移往服務導向架構時，通常需要容納現有的舊式系統，而這些系統執行企業的大部分作業，但是不具有服務功能。這些應用程式所使用的功能通常只有較新且已啟用服務的軟體套件才能存取，而且只能透過配接器、傳訊或其他非服務導向整合技術進行存取。

問題

舊式應用程式在本質上不符合服務導向架構 (SOA) 環境，而且變更為這類系統的成本十分昂貴，並且需要特殊的技能。此問題是尋找方法來整合這類系統與較新的服務導向套件及應用程式。尤其需要在下列兩者之間建立橋接器：一般與 Web 服務搭配使用的同步 HTTP 通訊協定，與舊式應用程式經常使用的傳訊通訊協定。

選擇指引

如果企業想要向要求端用戶端呈現服務介面，以作為 SOA 環境的一部分，但是無法輕鬆升級提供者應用程式以提供服務介面，則適合使用此型樣。

如果提供者應用程式提供 XML 介面，而用戶端應用程式支援對 Web 服務的呼叫，則使用此型樣。此型樣可以使用轉換加以延伸，以透過 WebSphere MQ 支援具有非 XML 介面的應用程式的服務 Facade。
