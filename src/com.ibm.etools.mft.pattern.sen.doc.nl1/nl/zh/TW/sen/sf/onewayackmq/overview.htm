Service Facade to WebSphere MQ: one-way with acknowledgment 型樣

使用 Service Facade to WebSphere MQ: one-way with acknowledgment 型樣，可以向用戶端呈現 Web 服務介面，以及使用已啟用 WebSphere MQ 的應用程式來滿足服務要求。

使用此型樣，可以橋接非同步 HTTP 通訊協定及可靠的傳訊通訊協定，以透過儲存要求以進行處理的安全來處理更新項目。

此型樣提供在計時、通訊協定及傳輸時，要求者與提供者之間的鬆散聯結。它適用於現有系統的服務介面。

解決方案

解決方案是實作可提供服務進入點的訊息流程。透過 HTTP 接收到 Web 服務要求時，會將資料作為持續訊息置入提供者應用程式的佇列中。放置訊息之後，會傳回確認通知，以確認已接受要求。如果用戶端接收到確認通知，則表示已將訊息寫入佇列。如果接收到錯誤，則不會傳送任何訊息，不過，如果錯誤是逾時，則此狀況是暫時的，但是用戶端應用程式會察覺到發生問題。
