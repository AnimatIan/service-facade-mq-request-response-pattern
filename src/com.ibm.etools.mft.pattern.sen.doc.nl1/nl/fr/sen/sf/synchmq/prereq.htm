Tâches à exécuter après avoir généré le modèle Façade de service sur WebSphere MQ : question-réponse

Avant d'appliquer le modèle pour produire une instance du modèle, vous devez déterminer les tâches à exécuter parmi les tâches suivantes. Si nécessaire, exécutez les tâches suivantes :

  • Vérifiez que le modèle répond aux besoins de l'entreprise et qu'il peut être pris en charge par les applications demandeur et fournisseur.
  • Définissez ou identifiez l'interface fournisseur et l'interface de service associée.
  • Affectez un nom à l'instance du modèle. Identifiez la fonction de l'application de modèle pour déterminer le nom approprié.
  • Sélectionnez les files d'attente et les gestionnaires de files d'attente WebSphere MQ. Vous pouvez utiliser les gestionnaires de files d'attente et les files d'attente existants, par exemple, pour accéder à l'application fournisseur et vous devez définir leurs noms.
  • Identifiez les gestionnaires de files d'attente et les files d'attente supplémentaires nécessaires au modèle et définissez leurs noms. Tenez compte des règles, des normes ou des conventions d'attribution de nom qui s'appliquent.
  • Déterminez si vous devez utiliser la consignation. Si la consignation est nécessaire, vous devez définir le gestionnaire de files d'attente et la file d'attente auxquels les messages de consignation sont envoyés. Vous pouvez envoyer les messages de consignation vers une file d'attente de consignation générale utilisée par plusieurs flux de messages ou utiliser une file d'attente spécifique pour l'instance du modèle.
  • Déterminez si vous avez besoin des messages d'erreur.
  • Déterminez si vous devez utiliser la validation.
  • Créez un fichier déployable WSDL dans un projet d'ensemble de messages.
