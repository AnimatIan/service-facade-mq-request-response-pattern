Paramètres de la Façade de service sur WebSphere MQ : modèle de demande-réponse

Informations de maintenance
Paramètre du modèle O C Valeur par défaut Description
Fichier WSDL de service Oui Non Non disponible Ce paramètre du modèle définit le fichier WSDL déployable qui définit l'interface de service présentée par le courtier aux demandeurs.

Cette valeur doit inclure le nom de chemin à partir du projet d'ensemble de messages dans l'espace de travail.

Pour accéder au fichier WSDL nécessaire, cliquez sur Parcourir.
Validation de demande SOAP Non Oui Aucun Les demandes de service entrantes sont validées conformément au paramètre. Valeurs acceptées :
  • Aucun
  • Contenu et valeur
Validation de la réponse SOAP Non Oui Aucun Les réponses SOAP sortantes sont validées conformément au paramètre. Valeurs acceptées :
  • Aucun
  • Contenu et valeur
Informations de réponse
Paramètre du modèle O C Valeur par défaut Description
File d'attente de réponses Oui Oui REPONSE Ce paramètre du modèle définit la file d'attente du courtier à laquelle l'application fournisseur doit envoyer les réponses.
File d'attente de stockage Oui Oui STORE Ce paramètre du modèle définit la file d'attente utilisée pour stocker les en-têtes MQMD lors de l'attente d'une réponse du fournisseur.
Expiration des messages de la file d'attente de stockage en dixièmes de seconde Oui Oui Pas de délai d'attente Ce paramètre du modèle définit le délai d'expiration en dixièmes de seconde des messages qui contiennent un identificateur de réponse HTTP.

Cette valeur ne doit pas être supérieure au délai de réponse maximum attendu.

Vous pouvez saisir votre propre valeur, ou accepter la valeur par défaut Pas de délai d'attente.
Informations sur les fournisseurs
Paramètre du modèle O C Valeur par défaut Description
Gestionnaire de files d'attente de fournisseur Non Oui   Ce paramètre du modèle définit le gestionnaire de files d'attente dans lequel l'application fournisseur reçoit les demandes du flux de messages du courtier.

Si vous ne le définissez pas, le gestionnaire de files d'attente du courtier est utilisé.
File d'attente de demandes de fournisseur Oui Oui FOURNISSEUR Ce paramètre du modèle définit la file d'attente dans laquelle l'application fournisseur reçoit les demandes du flux de messages du courtier.
Consignation
Paramètre du modèle O C Valeur par défaut Description
Consignation requise Non Non Effacé

Ce paramètre indique si l'instance de modèle inclut le code de consignation.

Si Consignation nécessaire est désélectionné, aucun flux secondaire Journal n'est inclus.

Si Consignation nécessaire est sélectionné, un flux secondaire Journal est inclus dans le flux de messages et vous devez définir les valeurs de Gestionnaire de files d'attente de consignation et File d'attente de journaux.
Gestionnaire de files d'attente de journaux Non Oui

Ce paramètre du modèle définit le gestionnaire de files d'attente de la consignation. Vous pouvez ne pas lui affecter une valeur si le gestionnaire de files d'attente du courtier est utilisé pour la consignation.

Vous devez le définir uniquement si vous sélectionnez Consignation nécessaire.

File d'attente de journaux Non Oui LOG

Ce paramètre définit la file d'attente de consignation.

Vous devez le définir uniquement si vous sélectionnez Consignation nécessaire.
Traitement d'erreurs
Paramètre du modèle O C Valeur par défaut Description
Message d'erreur nécessaire Non Non Sélectionné

Ce paramètre de modèle indique si l'application de modèle crée les éléments de flux de messages pour créer les messages d'erreur.

Si vous sélectionnez Message d'erreur nécessaire, vous devez définir les valeurs de Gestionnaire de file d'attente d'erreurs et File d'attente d'erreurs.
Gestionnaire de files d'attente d'erreurs Non Oui  

Ce paramètre de modèle définit le gestionnaire de files d'attente des messages d'erreur. Vous pouvez ne pas lui affecter une valeur si le gestionnaire de files d'attente du courtier est utilisé pour la consignation.
File d'attente des erreurs Non Oui ERROR

Ce paramètre du modèle définit la file d'attente des messages d'erreur.

Vous devez le définir uniquement si vous sélectionnez Message d'erreur nécessaire.

Dispositions générales
Paramètre du modèle O C Valeur par défaut Description
Schéma de courtier Non Non mqsi Vous pouvez utiliser un schéma de courtier pour créer un espace de nom pour l'instance du modèle.

Si vous définissez ce paramètre, tous les flux de messages et les fichiers ESQL sont créés dans le schéma de courtier nommé. Dans le cas contraire, le schéma par défaut est utilisé.

Préfixe de flux Non Non   Si vous définissez ce préfixe, il est ajouté au début de tous les noms de flux de messages.
Suffixe de flux Non Non   Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de flux de messages.
Préfixe de la file d'attente Non Non   Si vous définissez ce préfixe, il est ajouté au début de tous les noms de file d'attente.

Si vous utilisez les noms de file d'attente par défaut, vous devez définir le préfixe, le suffixe ou les deux pour identifier les noms de file d'attente de chaque instance du modèle.

Suffixe de la file d'attente Non Non   Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de file d'attente.
Description abrégée Non Non   Cette description est ajoutée à la propriété Description abrégée de tous les flux de messages générés.
Description détaillée Non Non   Cette description est ajoutée à la propriété Description détaillée de tous les flux de messages générés.

Tous les paramètres du modèle sont répertoriés avec leurs valeurs par défaut.

La colonne avec l'en-tête M indique si la propriété est obligatoire. La colonne avec l'en-tête C indique si la propriété est configurable, ce qui signifie que vous pouvez modifier la valeur lorsque vous ajoutez le flux de messages au fichier d'archive du courtier (BAR).

Lorsqu'aucune valeur n'est définie, la valeur par défaut est une chaîne vide.
