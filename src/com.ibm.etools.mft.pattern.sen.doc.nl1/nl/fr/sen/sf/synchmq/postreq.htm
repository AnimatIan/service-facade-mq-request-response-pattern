Tâches à exécuter après avoir généré le modèle Façade de service sur WebSphere MQ : demande-réponse

Tâches que vous devez exécuter après avoir généré le modèle.

  1. Créez le fichier archive de courtier (BAR). Le flux de messages doit être ajouté à un fichier archive de courtier. Les ensembles de messages référencés par le flux sont ajoutés automatiquement.
    1. Cliquez sur Fichier > Nouveau > Fichier BAR.
    2. Entrez le nom du projet de serveur ou sélectionnez-en un dans la liste affichée. La liste est filtrée pour contenir uniquement les projets dans l'ensemble actif de projets.
    3. Entrez le nom du fichier BAR à créer.
    4. Cliquez sur Terminer. Un fichier portant l'extension .bar est créé et s'affiche dans la vue Explorateur d'administration du courtier sous le dossier Archives de courtier. L'éditeur de contenu du fichier BAR s'affiche.
    5. Dans l'onglet Préparation, sélectionnez le fichier de flux de messages généré et ajoutez-le au fichier BAR.
    6. Sélectionnez Compiler et intégrer des ressources et cliquez ensuite sur Générer et sauvegarder pour créer le fichier BAR.
  2. Configurez le fichier archive du courtier. Il n'est pas nécessaire d'effectuer d'autres opérations de configuration au niveau du flux de messages dans le fichier archive de courtier avant de déployer les flux de messages de l'instance du modèle Façade de service sur WebSphere MQ : demande-réponse. Néanmoins, vous pouvez vérifier certaines options par défaut et configurer des modifications pour votre environnement. Dans la page Gestion de l'éditeur d'archive de courtier :
    1. Facultatif : utilisez la propriété RequestLoggingOn définie par l'utilisateur pour activer ou désactiver le flux de messages Demande.
    2. Facultatif : utilisez la propriété ResponseLoggingOn définie par l'utilisateur pour activer ou désactiver le flux de messages Réponse.
    3. Facultatif : utilisez la propriété ErrorLoggingOn définie par l'utilisateur pour indiquer si des messages d'erreur sont générés.
    4. Facultatif : vous pouvez modifier les propriétés configurables dans l'archive de courtier, telles que les propriétés de sécurité ou de création d'instances supplémentaires.

      Vous devez vérifier que les valeurs que vous changez n'affectent pas le comportement de l'instance du modèle.

  3. Créez les files d'attente. Lorsque vous appliquez le modèle, la liste des files d'attente WebSphere MQ est générée, avec des noms de file d'attente et de gestionnaire de file d'attente, que vous devez créer s'ils n'existent pas. Tous les paramètres de message nécessaires au modèle sont définis explicitement dans les messages et ne dépendent pas des paramètres de file d'attente.
    1. Dans le fichier récapitulatif, consultez la liste des files d'attente que vous devez créer. Ce fichier se trouve dans le dossier Configuration du modèle du projet d'instance de modèle InstanceName.
    2. Créez les ressources WebSphere MQ. Vous pouvez créer les ressources WebSphere MQ depuis la ligne de commande en utilisant des programmes et des utilitaires, tels que runmqsc, ou en utilisant l'interface graphique WebSphere MQ Explorer. Pour plus d'informations sur la création des ressources WebSphere MQ, voir WebSphere MQ Intercommunication sur la page Web Bibliothèque WebSphere MQ.
  4. Déployez le fichier BAR. L'ensemble de messages qui contient le fichier déployable WSDL doit être déployé en utilisant le même ou un autre fichier BAR dans chaque courtier où les flux sont déployés.
