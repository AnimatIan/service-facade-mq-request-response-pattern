Contraintes d'utilisation du modèle Façade de service sur WebSphere MQ : question-réponse

Il existe certaines contraintes lorsque vous utilisez le modèle.

Vous ne pouvez utiliser le modèle Façade de service sur WebSphere MQ : question-réponse que si toutes les contraintes suivantes sont respectées :

  • Les demandes doivent être reçues sous forme de messages SOAP sur HTTP.
  • Les applications demandeurs doivent prendre en charge un style d'interaction réponse-demande synchrone.
  • Les applications fournisseurs doivent pouvoir répondre aux demandes dans un délai approprié pour l'interaction synchrone.
  • Les applications fournisseurs peuvent recevoir les demandes et retourner les réponses sur WebSphere MQ.
  • Les applications fournisseurs doivent prendre en charge le comportement standard de copie de l'identificateur du message dans l'identificateur de corrélation.
  • Les fichiers WSDL qui définissent l'interface de courtier disposent d'un seul port et d'une seul liaison.
  • Les fichiers WSDL doivent contenir les définitions qui spécifient les propriétés du noeud SOAPInput dans les définitions du fichier WSDL de niveau supérieur. Par conséquent, certains fichiers WSDL imbriqués complexes ne peuvent pas être traités automatiquement par le modèle.

    Si des fichiers WSDL qui ne respectent pas ces contraintes sont utilisés, le modèle ne configure pas le noeud SOAPInput, mais il liste la configuration du noeud sous la forme d'une tâche à exécuter après avoir généré l'instance du modèle.

  • L'application fournisseur prend en charge les messages WebSphere MQ XML dans un format qui peut être utilisé comme corps d'une demande SOAP ou vous pouvez personnaliser le flux secondaire Traitement de la demande pour convertir le format en messages non-XML.
  • Si la consignation est nécessaire, le format utilisé par le modèle est supposé répondre aux conditions de votre environnement.
