Flux secondaire Journal pour le modèle Façade de service sur WebSphere MQ : question-réponse

La consignation des messages dans une base de données n'intervient pas dans les flux de messages qui gèrent les transactions métier. Des messages persistants sont écrits dans une file d'attente WebSphere MQ qui est traitée séparément. L'écriture des messages persistants dans une file d'attente WebSphere MQ isole l'impact des mises à jour de la base de données sur les performances, ce qui permet de contrôler où et quand les files d'attente sont traitées.

Le modèle suppose que les données de journal sont ajoutées à l'en-tête MQRFH2 s'il en existe un. Si aucun en-tête MQRFH2 n'existe, un en-tête est créé. Le corps du message est consigné à son arrivée dans le flux des messages. Les données de journal suivantes sont ajoutées :

  • Horodatage du journal
  • Nom du courtier
  • Nom du flux de messages
  • Etat ( 0 succès)
  • Mode - Demande/Réponse
