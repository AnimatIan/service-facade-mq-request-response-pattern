Flux secondaire Réponse pour le modèle Façade de service sur WebSphere MQ : question-réponse

Le flux Réponse est exécuté sous la forme d'un flux non transactionnel.

Lorsqu'une réponse est reçue de l'application fournisseur, les actions suivantes sont exécutées :

  1. Le flux de réponse lit le message de réponse provenant du fournisseur. La validation est déterminée par les paramètres du modèle.
  2. Le flux utilise l'ID de corrélation pour récupérer l'Identifiant de réponse SOAP auprès de la file d'attente WebSphere MQ interne. L'Identifiant de réponse SOAP est ajouté à l'environnement local afin de contrôler la destination de la réponse SOAP.
  3. Un en-tête de réponse HTTP est créé.
  4. Si une exception se produit et que la consignation d'erreur est incluse dans l'application du modèle, le flux secondaire Erreur est appelé, et une erreur est écrite dans la file d'attente de réponse. Cependant, dans la mesure où la défaillance implique que l'IDdeRéponse qui devait répondre à la demande SOAP n'est pas disponible, aucune tentative n'est faite de répondre au demandeur.
  5. Une réponse réussie n'est consignée que si la consignation est incluse dans cette application du modèle et activée.
