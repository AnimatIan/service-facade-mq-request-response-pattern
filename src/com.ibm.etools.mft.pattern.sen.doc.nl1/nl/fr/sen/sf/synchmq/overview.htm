Modèle Façade de service sur WebSphere MQ : demande-réponse

Utilisez le modèle Façade de service sur WebSphere MQ : demande-réponse pour fournir une façade de service Web aux fonctions accessibles uniquement via WebSphere MQ. Ce modèle établit un pont entre le protocole HTTP synchrone, généralement utilisé avec les services Web, et les applications existantes avec des interfaces WebSphere MQ qui ne peuvent pas être mises à niveau aisément.

Utilisez ce modèle lorsque les applications fournisseurs mettent à disposition une interface XML et que les applications client prennent en charge les appels vers les services Web. Vous pouvez étendre le modèle avec des transformations pour prendre en charge une façade de service dans les applications avec une interface non-XML sur WebSphere MQ.

Solution

La solution consiste à implémenter un flux de messages qui fournit un point d'entrée de service. Lorsqu'une demande de service est reçue, l'identificateur de réponse est stocké dans une file d'attente interne et la demande est envoyée à l'application demandeur. Lorsque l'application fournisseur répond, l'identificateur de la réponse est récupéré et une réponse des services Web, qui contient la réponse du fournisseur, est envoyée à l'application demandeur.
