Modèle Façade de service sur WebSphere MQ : unidirectionnel avec accusé de réception

Utilisez le modèle Façade de service vers WebSphere MQ : unidirectionnel avec accusé de réception pour présenter une interface de service Web aux clients et répondre aux demandes de service en utilisant une application WebSphere MQ.

Utilisez ce modèle pour relier les protocoles HTTP asynchrones et les protocoles de messagerie fiables pour gérer les mises à jour en enregistrant les demandes pour les traiter.

Ce modèle fournit un couplage large entre les demandeurs et les fournisseurs en terme de timing, de protocoles et de transport. Il est recommandé que le service s'interface avec les systèmes existants.

Solution

La solution consiste à implémenter un flux de messages qui fournit un point d'entrée de service. Lorsqu'une demande de service Web est reçue sur HTTP, les données sont placées sous la forme d'un message persistant dans une file d'attente à l'attention de l'application fournisseur. Lorsque le message est placé, un accusé de réception est envoyé pour confirmer que la demande a été acceptée. Si un client reçoit un accusé de réception, le message est placé dans la file d'attente. Si une erreur se produit, aucun message n'est envoyé. S'il s'agit d'un dépassement de délai d'attente, le cas est indéterminé, mais les applications client savent qu'un incident s'est produit.
