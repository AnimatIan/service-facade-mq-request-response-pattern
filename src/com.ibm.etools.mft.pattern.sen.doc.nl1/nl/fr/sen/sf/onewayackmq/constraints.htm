Contraintes d'utilisation du modèle Façade de service vers WebSphere MQ : unidirectionnel avec accusé de réception

Il existe certaines contraintes lorsque vous utilisez le modèle.

Vous ne pouvez utiliser le modèle Façade de service sur WebSphere MQ : unidirectionnel avec accusé de réception que si toutes les contraintes suivantes sont respectées :

  • Les demandes sont reçues sous forme de messages SOAP sur HTTP qui utilise un style littéral de document.
  • Les demandes et les réponses SOAP utilisent le CCSID 1208.
  • Les applications demandeurs peuvent prendre en charge le modèle Façade de service vers WebSphere MQ : unidirectionnel avec accusé de réception.
  • Les applications fournisseurs sont chargées d'agir sur une mise à jour lorsqu'un message est envoyé à sa file d'attente d'entrée.
  • Les fichiers WSDL qui définissent l'interface du courtier ne disposent que d'un seul port et d'une seule liaison.
  • Les fichiers WSDL doivent contenir les définitions qui spécifient les propriétés du noeud SOAPInput dans les définitions du niveau supérieur WSDL. Par conséquent, certains fichiers WSDL imbriqués complexes ne peuvent pas être traités automatiquement par le modèle.

    Si des fichiers WSDL qui ne respectent pas ces conditions sont utilisés, le modèle ne configure pas le noeud SOAPInput, mais il liste la configuration du noeud sous la forme d'une tâche à exécuter après avoir généré l'instance du modèle.

  • L'application fournisseur prend en charge les messages WebSphere MQ XML dans un format qui peut être utilisé comme corps d'une demande SOAP ou vous pouvez personnaliser le flux secondaire Traitement de la demande pour convertir le format en message non-XML.
  • Le modèle définit son propre format pour les messages de consignation et d'erreur et ces messages sont supposés être suffisants pour répondre aux besoins de votre environnement.

Certaines des contraintes précédentes, notamment le CCSID, peuvent être éliminées après avoir appliqué le modèle.
