Tâches à exécuter après avoir généré le modèle Façade de service sur WebSphere MQ : unidirectionnel avec accusé de réception

Tâches que vous devez exécuter après avoir généré le modèle.

  1. Vérifiez le CCSID. Ce modèle suppose que le CCSID utilisé pour les demandes SOAP est 1208. Si l'application fournisseur nécessite des messages avec un CCSID, vous devez définir ce paramètre dans le flux secondaire Processeur des demandes en utilisant l'éditeur de flux de messages. Cette propriété ne peut pas être modifiée avec l'éditeur d'archive de courtier.
  2. Vérifiez la configuration du noeud SOAP. Dans le cas de fichiers WSDL complexes, il peut être nécessaire de sélectionner le fichier WSDL dans l'éditeur de flux de messages, ce qui est indiqué dans le fichier récapitulatif.
  3. Créez le fichier archive de courtier (BAR). Ajoutez le flux de messages généré au fichier BAR. Dans l'éditeur d'archives du courtier, vous devez sélectionner l'option Compiler et intégrer des ressources avant de générer le fichier BAR. Vous devez également ajouter les ensembles de messages auxquels le flux fait référence.
  4. Configurez le fichier archive du courtier. Il n'est pas nécessaire d'effectuer d'autres opérations de configuration au niveau du flux de messages dans le fichier archive de courtier avant de déployer les flux de messages de l'instance du modèle Façade de service sur WebSphere MQ : unidirectionnel avec accusé de réception. Néanmoins, vous pouvez vérifier certaines options par défaut et configurer des modifications pour votre environnement. Par exemple, vous pouvez vérifier les propriétés suivantes :
    • LoggingOn. Si la consignation est incluse, lors du déploiement, cette option détermine si la consignation des demandes est nécessaire pour l'environnement en cours.
    • ErrorLoggingOn. Si les messages d'erreur sont inclus, lors du déploiement, cette option détermine si la consignation des erreurs est nécessaire pour l'environnement en cours.
    Vous pouvez changer les paramètres du modèle qui répertoriés dans les table de paramètres comme étant définissables, mais vous devez faire preuve de vigilance, car le modèle peut utiliser par défaut un comportement spécifique. Toutefois, vous pouvez changer les propriétés définies par des caractéristiques de fonctionnement qui n'affectent pas la logique de flux (configuration de la sécurité ou instances supplémentaires, par exemple).
  5. Créez les files d'attente. Lorsque vous appliquez le modèle, la liste des files d'attente WebSphere MQ est générée, avec les noms de file d'attente et de gestionnaire de file d'attente, que vous devez créer s'ils n'existent pas. Tous les paramètres de message nécessaires au modèle sont définis explicitement dans les messages et ne dépendent pas des paramètres de file d'attente.
    • Dans le fichier récapitulatif, consultez la liste des files d'attente que vous devez créer. Ce fichier se trouve dans le dossier Configuration du modèle du projet d'instance de modèle InstanceName.
    • Créez les ressources WebSphere MQ. Vous pouvez créer les ressources WebSphere MQ depuis la ligne de commande en utilisant des programmes et des utilitaires, tels que runmqsc, ou en utilisant l'interface graphique WebSphere MQ Explorer. Pour plus d'informations sur la création des ressources WebSphere MQ, voir WebSphere MQ Intercommunication sur la page Web Bibliothèque WebSphere MQ.
  6. Déployez le fichier BAR. L'ensemble de messages qui contient le fichier déployable WSDL doit être déployé en utilisant le même ou un autre fichier BAR dans chaque courtier où les flux sont déployés.
