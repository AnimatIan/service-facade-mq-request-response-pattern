Flux secondaire Erreur pour le modèle Façade de service sur WebSphere MQ : unidirectionnel avec accusé de réception

Ce flux secondaire teste si un message d'erreur doit être écrit, ce qui est déterminé par la propriété ErrorLoggingOn définie par l'utilisateur.

  1. Si vous avez configuré les paramètres du modèle pour inclure des messages d'erreur et que la propriété définie par l'utilisateur, ErrorLoggingOn est sélectionnée, un message d'erreur est rédigé et s'affiche dans la file d'attente des erreurs.

    Les détails de l'exception sont écrits sous forme de message XML. Ces détails comprennent :

    • Nom du courtier
    • Nom du flux
    • Horodatage
    • Récapitulatif des données de l'exception

    Le récapitulatif des données d'exception contient le libellé du flux de messages, le numéro d'erreur, la description de texte, ainsi qu'une liste d'insertions donnant plus de détails.

  2. Si le message au fournisseur a été rédigé et qu'une réponse est envoyée au demandeur, aucune autre action n'est effectuée.
  3. Si le message au fournisseur n'a pas été rédigé, une exception est générée qui provoque le renvoi d'un incident SOAP au demandeur.
